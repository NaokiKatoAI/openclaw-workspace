## 🔍 SEO/AI対策マップ

### 構造化データ（Schema.org）

| ページ | スキーマ | 目的 |
|--------|---------|------|
| 全商品詳細 | `Product` | 検索結果にリッチスニペット（価格・在庫） |
| /faq/ + 商品内FAQ | `FAQPage` | AIが引用しやすいQ&A形式 |
| 全ページ | `BreadcrumbList` | サイト構造の明示 |
| TOP | `Organization` + `WebSite` | ブランド認知・サイトリンク |
| /howto/ | `HowTo` | コンタクト使い方のリッチ表示 |
| /products/{slug}/ | `Review` / `AggregateRating` | レビュー星表示 |

### テキストコンテンツ化（現状: ほぼ全て画像）

| 対象 | 現状 | Ph2対応 |
|------|------|---------|
| ナビゲーション | SVG/PNG画像 | テキスト＋CSSスタイリング |
| 商品タイトル | 画像 | HTML `<h1>`〜`<h3>` |
| 商品スペック | 画像内テキスト | HTMLテーブル/リスト |
| ボタン | 画像 | CSS `<button>` / `<a>` |
| バナー | 画像のみ | 画像 + alt属性 + テキスト併記 |

### AI対策（ゼロクリック対応）

| 施策 | 内容 | 優先度 |
|------|------|--------|
| note運用 | 第三者目線の記事（「シリコーンコンタクトの選び方」等） | 高 |
| FAQ充実 | AIが引用しやすい「Q: ○○ A: ○○」形式 | 高 |
| 定義系コンテンツ | 「リフレアとは」「シリコーンコンタクトとは」 | 中 |
| 見出し階層最適化 | h1→h2→h3の正しい階層構造 | 高 |
| alt属性最適化 | 全画像に適切なalt | 高 |

### 内部リンク設計

```
TOP（診断結果）──→ 商品詳細
TOP（3タブ）────→ 商品詳細 / カテゴリ一覧
TOP（定期入口）──→ /teiki/
商品詳細 ────────→ 関連商品 / 定期便 / FAQ
/faq/ ──────────→ 商品詳細 / /guide/ / /howto/
/howto/ ─────────→ 商品詳細 / /faq/
/about/ ─────────→ 商品一覧 / 診断
/words/ ─────────→ 商品詳細（該当用語の商品）
note記事 ────────→ 商品詳細 / TOP
```
