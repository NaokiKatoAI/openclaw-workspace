# 案件ステータス管理システム

## 概要
各クライアントの案件進捗をNotionで一元管理するシステム。
Discordの各クライアントチャンネルで状況報告すると、自動でNotionに記録される。

## ワークフロー

```
1. 案件作成
   ↓
2. 見積もり作成済み
   ↓
3. 見積もり送付済み (クラウドサイン送付)
   ↓
4. 見積もり承認済み (返送完了 → 作業開始OK)
   ↓
5. 作業中
   ↓
6. 作業完了 (先方確認依頼)
   ↓
7. 先方OK
   ↓
8. 検収書作成済み
   ↓
9. 検収書送付済み (クラウドサイン送付)
   ↓
10. 検収書承認済み (返送完了 → 納品完了)
   ↓
11. 請求書発行済み
   ↓
12. 完了
```

## 使い方

### クライアントチャンネルでの報告例

**かっぴー:** 「見積もり送った」
**ガッツ:** 「了解、Notionに記録したぜ。ステータス：見積もり送付済み → 見積もり承認待ち」

**かっぴー:** 「検収書返ってきた」
**ガッツ:** 「OK、納品完了だな。次は請求書発行だぜ」

**かっぴー:** 「今どうなってる？」
**ガッツ:** 「現在のステータス：検収書承認済み。次は請求書発行だ」

## 認識キーワード

- **見積もり送った** → 見積もり送付済み
- **見積もり承認** / **見積OK** → 見積もり承認済み
- **作業開始** → 作業中
- **作業完了** / **完了した** → 作業完了
- **先方OK** / **OKもらった** → 先方OK
- **検収書送った** → 検収書送付済み
- **検収書返ってきた** / **検収書承認** → 検収書承認済み
- **請求書送った** / **請求書発行** → 請求書発行済み

## Notionへの記録フォーマット

```
【現在の案件】
案件名: 〇〇サイト制作
ステータス: 見積もり送付済み
最終更新: 2026-02-03
次のアクション: 見積もり承認待ち
```

## スクリプト

### ステータス更新
```bash
node scripts/project-status.js update <channelId> <ステータス> [案件名]
```

### ステータス確認
```bash
node scripts/project-status.js get <channelId>
```

## 実装メモ

- 各クライアントのDiscordチャンネルでキーワードを検知
- 自動でNotionページに案件状況を追加
- 金額・詳細情報は各クライアントチャンネルで個別管理
- ここ（#ガッツとの会話）では全体フロー管理のみ
